THE GIST

Brains are stores. Git is history.

A brain is just a directory of markdown files with enough structure to be useful: names, categories, tags, paths, writable rules, and a memory index that can answer real questions. Git gives those thoughts a timeline. Search gives them retrieval. The result is memory that feels inspectable instead of mystical.

your mind git history

ORGANIZATION

Brains are not a blob.

grug gives you organizational primitives: categories, paths, tags, backlinks, and the idea of separate brains. You can split research from project docs, writable memory from synced archives, or personal notes from machine-readable reference.

SEARCH

Context stays queryable.

The agent can search all brains at once, read specific memories, recall a brain summary, and traverse local relationships. It does not need to pretend it remembers, it can actually look.

WHAT IT PROVIDES

  • 01 Local-first markdown storage with explicit brain boundaries.
  • 02 Search over every memory, not just the one you happened to open.
  • 03 Read, write, rename, delete, sync, and recall operations for agents.
  • 04 A structure that survives between sessions and across machines.

FOR THE AGENT

What grug does for your agent is simple: it makes context durable.

Instead of treating every session like a reset, the agent gets a place to put facts, plans, feedback, and work products. Instead of hoping the right chunk is still in the prompt window, it gets a memory system with stable paths and retrieval hooks. Instead of swallowing conflicts, it can surface them and keep the trail.

TIME MATTERS

Memory gets more useful when you can trust its history.

grug keeps the operational shape of version control: sync timers, conflict files, branches, history, and the ability to inspect what changed. That matters because agent memory is not just storage, it is a record of evolving understanding.

THE SHORT VERSION

grug-brain is memory that behaves like infrastructure.

Local by default. Searchable. Organized. Versioned. Syncable. The agent gets a real store with real history, which means context can accumulate without becoming opaque.
